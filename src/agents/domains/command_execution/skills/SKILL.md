# 电力储能指令执行标准作业程序 (SOP)

## 1. 概述

本技能定义了电力储能系统指令执行的标准作业程序和安全协议，确保所有操作安全、合规、可追溯。

**适用范围**：所有电力储能设备的控制指令执行

**核心目标**：
- 确保设备安全运行
- 防止误操作和安全事故
- 提供完整的审计追踪
- 建立标准化操作流程

## 2. 核心约束与规则

### 2.1 安全阈值

- **规则1**：单次充放电功率不得超过设备额定功率的110%
- **规则2**：SOC运行范围应保持在10%-95%之间，极端情况不低于5%、不高于98%
- **规则3**：设备温度超过50℃时禁止大功率操作，超过55℃时禁止任何操作
- **规则4**：电压运行范围：300V-450V，建议范围：350V-420V
- **规则5**：电流运行范围：-1000A至1000A（负值表示放电）

### 2.2 操作权限

- **规则6**：所有操作必须通过参数验证、安全检查、权限验证三重审查
- **规则7**：高风险操作（功率>1000kW、风险等级≥高）必须二次确认
- **规则8**：设备故障或离线状态下严禁执行任何操作指令
- **规则9**：关键操作（复位、清零、固件升级）仅限管理员权限

### 2.3 时序约束

- **规则10**：充电和放电状态之间必须有5秒以上的间隔（先停止再切换）
- **规则11**：同一设备在10秒内只能接受一条有效指令
- **规则12**：功率变化速率不得超过500kW/min

### 2.4 设备保护

- **规则13**：SOH（健康度）低于80%的设备限制充放电功率至50%
- **规则14**：SOC低于15%时禁止放电，SOC高于90%时禁止充电
- **规则15**：连续3次操作失败后暂停该类操作，建议人工介入

## 3. 标准化工作流 (SOP)

当你接到执行指令的任务时，请严格遵循以下流程：

### 第一步：指令审查（必须）

调用工具：`review_and_validate_command`

**检查项**：
- ✅ 参数合法性（类型、范围、完整性）
- ✅ 设备状态兼容性
- ✅ 操作权限验证
- ✅ 时序冲突检测

**决策**：
- ✅ 通过 → 进入第二步
- ❌ 未通过 → 返回拒绝原因，停止流程

### 第二步：风险评估（必须）

调用工具：`assess_command_risk`

**分析维度**：
- 设备健康度影响
- 经济影响
- 安全风险
- 操作复杂度

**决策**：
- 低/中风险 → 可以直接执行
- 高风险 → 建议二次确认
- 严重风险 → 必须明确确认，否则拒绝

### 第三步：执行决策（根据审查结果）

**场景A**：审查通过 + 低/中风险
- 直接执行，无需确认

**场景B**：审查通过 + 高风险
- 建议用户二次确认
- 说明风险因素
- 等待确认后再执行

**场景C**：审查通过 + 严重风险
- 必须请求用户明确确认
- 提供详细风险分析
- 建议替代方案

**场景D**：审查未通过
- 拒绝执行
- 返回详细原因
- 提供改进建议

### 第四步：执行指令（仅当审查通过）

调用工具：`execute_device_command`

**执行步骤**：
1. 建立设备通信连接
2. 下发指令到设备
3. 监控执行状态
4. 处理执行结果

**异常处理**：
- 执行失败 → 记录错误，分析原因
- 通信超时 → 重试3次，失败后报告
- 设备故障 → 立即停止，报告状态

### 第五步：记录日志（必须）

调用工具：`log_operation`

**记录内容**：
- 操作ID
- 操作类型
- 设备ID
- 操作者
- 执行状态
- 关键参数
- 执行结果
- 时间戳

### 第六步：异常处理（如需要）

调用工具：`emergency_stop_all`

**触发条件**：
- 检测到严重安全风险
- 设备异常状态
- 接到紧急停止指令
- 操作者主动取消

## 4. 安全阈值参考表

| 参数 | 最小值 | 最大值 | 建议范围 | 说明 |
|------|--------|--------|----------|------|
| 功率 (kW) | 0 | 5000 | 100-1000 | 不得超过额定功率110% |
| SOC (%) | 5 | 98 | 20-90 | 极端情况可超出，但需人工确认 |
| 温度 (℃) | -10 | 55 | 15-45 | >50℃禁止大功率，>55℃禁止操作 |

| SOH (%) | 0 | 100 | >90 | <80%限制功率至50% |

## 5. 风险等级标准

### 低风险
- 功率 < 500kW
- SOC在20%-80%之间
- 温度在15-35℃之间
- SOH > 90%
- 常规操作时间

### 中风险
- 功率 500-1000kW
- SOC在10%-20%或80%-90%之间
- 温度在35-45℃之间
- SOH在80%-90%之间
- 非常规操作时间

### 高风险
- 功率 1000-2000kW
- SOC在5%-10%或90%-95%之间
- 温度在45-50℃之间
- SOH < 80%
- 状态切换操作

### 严重风险
- 功率 > 2000kW
- SOC < 5%或 > 95%
- 温度 > 50℃
- 设备有故障历史
- 非常规操作组合

## 6. 常见场景处理

### 场景1：充电指令

**审查要点**：
- 检查当前SOC是否已接近上限
- 验证充电功率是否在允许范围
- 确认设备是否处于可充电状态

**执行步骤**：
1. 查询设备状态
2. 审查充电参数
3. 评估风险
4. 执行充电指令
5. 监控充电进度
6. SOC达到目标后停止

**注意事项**：
- SOC > 90%时建议降低功率
- 充电过程中每小时检查温度
- 达到目标SOC前5%时准备停止

### 场景2：放电指令

**审查要点**：
- 检查当前SOC是否足够放电
- 验证放电功率是否在允许范围
- 确认设备是否处于可放电状态

**执行步骤**：
1. 查询设备状态
2. 审查放电参数
3. 评估风险
4. 执行放电指令
5. 监控放电进度
6. SOC达到下限时停止

**注意事项**：
- SOC < 20%时建议降低功率
- 放电过程中每小时检查温度
- 达到最低SOC前5%时准备停止

### 场景3：状态切换

**审查要点**：
- 检查当前状态是否允许切换
- 验证是否有足够的间隔时间
- 确认是否需要先停止

**执行步骤**：
1. 先停止当前操作
2. 等待5秒间隔
3. 查询设备状态确认已停止
4. 执行新的操作指令

**注意事项**：
- 切换前必须先停止
- 不要直接从充电切换到放电
- 确保间隔时间充足

### 场景4：紧急停止

**触发条件**：
- 检测到严重安全风险
- 设备异常状态
- 接到紧急停止指令

**执行步骤**：
1. 立即调用 `emergency_stop_all`
2. 不等待任何确认
3. 记录停止原因
4. 通知相关人员

**注意事项**：
- 紧急停止无需确认
- 停止后需要人工确认才能恢复
- 详细记录停止原因供后续分析

## 8. 质量检查清单

在执行任何指令前，确认以下检查项：

- [ ] 参数验证通过（类型、范围、完整性）
- [ ] 设备状态兼容（无故障、无离线）
- [ ] 操作权限足够
- [ ] 无时序冲突
- [ ] 安全阈值合规
- [ ] 风险评估完成
- [ ] 操作日志准备就绪
- [ ] 应急预案明确

## 9. 异常处理预案

### 通信失败
- 重试3次
- 检查网络连接
- 确认设备在线
- 失败后报告并记录

### 设备故障
- 立即停止所有操作
- 记录故障代码
- 通知维护人员
- 暂停后续操作

### 参数超限
- 拒绝执行指令
- 提供参数调整建议
- 说明安全风险
- 等待参数修正

### 操作失败
- 分析失败原因
- 记录错误信息
- 提供解决方案
- 决定是否重试

## 10. 持续改进

本SOP应根据实际运行情况持续改进：

- 分析异常模式
